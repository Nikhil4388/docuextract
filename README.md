## в—Ҳ WHAT IS THIS?

**MultiPDFToExcel** is a next-generation AI document intelligence platform that eliminates manual data entry from PDFs. Point it at any folder of PDFs вҖ” invoices, resumes, contracts, bank statements, medical records вҖ” define what fields you want, and receive a perfectly structured Excel spreadsheet in seconds.

No OCR setup. No templates to train. No code to write. Just upload and extract.

```
вҖў Future providers: Microsoft, Apple (on roadmap).

JWT Token Architecture
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
вҖў access_token  вҶ’ stored in JS memory (Zustand store) ONLY.
                  Never in localStorage, sessionStorage, or cookies.
                  Invisible to browser extensions and XSS payloads.
                  Expires: 60 minutes.
вҖў refresh_token вҶ’ stored in localStorage for session persistence.
                  Used only to silently re-issue access tokens.
                  Expires: 30 days.
вҖў Silent rotation: Axios interceptor queues concurrent requests during
                   token refresh, preventing race conditions.
вҖў On 401: custom event dispatched (not window.location.href redirect)
          to avoid hard-page-reload auth loops on new devices.
```

### 2 вҖ” Data Lifecycle (72-Hour Auto-Delete)

```
Upload вҶ’ S3 storage (AES-256 SSE)
       вҶ’ Celery extraction job runs
       вҶ’ Results stored in PostgreSQL
       вҶ’ 72-hour deletion timer starts
       вҶ’ All PDFs and raw data permanently deleted from S3 + DB
       вҶ’ No backups of user documents retained beyond this window
```

- We do **not** sell, share, or analyze your documents for any other purpose.
- Deletion is automatic and unconditional вҖ” no action required from you.
- PostgreSQL result rows are pruned on the same 72-hour schedule.

### 3 вҖ” Encryption

```
In Transit:
  вҖў TLS 1.3 enforced end-to-end (browser вҶ” Vercel, browser вҶ” Railway)
  вҖў HSTS header: Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
  вҖў Upstash Redis: TLS-only connections (no plaintext Redis)
  вҖў Supabase PostgreSQL: SSL-only, reject if no SSL

At Rest:
  вҖў AWS S3: SSE-S3 (AES-256) вҖ” each object encrypted with a unique key
  вҖў PostgreSQL credentials: AES-encrypted via ENCRYPTION_KEY before storage
  вҖў Passwords: bcrypt (12 rounds) вҖ” auto-rehash if cost factor increases
  вҖў Google OAuth users: zero password stored (OAuth token only)
```
